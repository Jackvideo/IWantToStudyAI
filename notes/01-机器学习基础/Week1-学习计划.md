### Day 1（周一）：机器学习是什么？
**时长**：60 分钟 | **对应**：西瓜书 1.1 节

#### 🎬 学习内容
| 时间段 | 内容 | 资源 |
|--------|------|------|
| 25分钟 | 观看视频 | [BV1gG411f7zX](https://www.bilibili.com/video/BV1gG411f7zX/) P1 第1章绪论部分 |
| 25分钟 | 阅读教材 | 西瓜书 1.1 节「引言」(第1-2页) |
| 10分钟 | 记录笔记 | 写下对机器学习的理解 |

#### 📝 核心要点（西瓜书 1.1）
```
机器学习的定义（Mitchell, 1997）：
"假设用 P 来评估计算机程序在某任务类 T 上的性能，
若一个程序通过利用经验 E 在 T 中任务上获得了性能改善，
则我们就说关于 T 和 P，该程序对 E 进行了学习。"

┌─────────────────────────────────────────────────────────┐
│  简化理解：                                             │
│  机器学习 = 从【经验/数据】中学习，提升【任务性能】      │
│                                                         │
│  西瓜例子：                                             │
│  - 经验 E：看过的西瓜（颜色、声音、纹理等）             │
│  - 任务 T：判断西瓜好不好吃                             │
│  - 性能 P：判断的准确率                                 │
└─────────────────────────────────────────────────────────┘

为什么叫"西瓜书"？
└─ 全书用"挑选好瓜"作为主要例子贯穿始终
```

### Day 2（周二）：基本术语
**时长**：60 分钟 | **对应**：西瓜书 1.2 节

#### 🎬 学习内容
| 时间段 | 内容 | 资源 |
|--------|------|------|
| 25分钟 | 观看视频 | 视频 P1 基本术语部分 |
| 25分钟 | 阅读教材 | 西瓜书 1.2 节「基本术语」(第2-5页) |
| 10分钟 | 制作术语卡片 | 整理核心术语 |

#### 📝 核心要点（西瓜书 1.2）
```
🔤 数据相关术语

【数据集 Dataset】D = {x₁, x₂, ..., xₘ}
└─ 一组记录的集合，共 m 个样本

【样本/示例 Sample/Instance】
└─ 一条记录，如一个西瓜的描述

【属性/特征 Attribute/Feature】
└─ 描述样本的某个方面，如"色泽"、"根蒂"

【属性值 Attribute Value】
└─ 属性的取值，如"青绿"、"蜷缩"

【属性空间/样本空间/输入空间】
└─ 所有属性张成的空间

【特征向量 Feature Vector】
└─ 样本在属性空间中的表示

📊 西瓜数据集示例（表 1.1）
┌─────┬────────┬────────┬────────┬────────┐
│编号 │  色泽  │  根蒂  │  敲声  │  好瓜  │
├─────┼────────┼────────┼────────┼────────┤
│  1  │  青绿  │  蜷缩  │  浊响  │   是   │
│  2  │  乌黑  │  蜷缩  │  沉闷  │   是   │
│  3  │  浅白  │  硬挺  │  清脆  │   否   │
│ ... │  ...   │  ...   │  ...   │  ...   │
└─────┴────────┴────────┴────────┴────────┘

特征向量示例：x₁ = (青绿, 蜷缩, 浊响)
```

```
🎯 学习相关术语

【标记 Label】
└─ 样本的"结果"信息，如"好瓜"

【样例 Example】= (xᵢ, yᵢ)
└─ 有标记的样本

【训练/学习 Training/Learning】
└─ 从数据中学得模型的过程

【训练数据 Training Data】
└─ 训练时使用的数据

【训练样本 Training Sample】
└─ 训练数据中的每个样本

【假设 Hypothesis】
└─ 学得的模型，对应数据潜在规律

【真相/真实 Ground-truth】
└─ 数据的真实规律

【学习器 Learner】
└─ 学习算法在数据上的实例化
```

```
📂 学习任务分类

【分类 Classification】
├─ 预测离散值（类别）
├─ 二分类：好瓜/坏瓜
└─ 多分类：猫/狗/鸟/...

【回归 Regression】
├─ 预测连续值
└─ 例：预测西瓜的成熟度（0~1）

【聚类 Clustering】
├─ 无监督学习
├─ 将数据分成若干组（簇）
└─ 例：把西瓜分成几个品种

【监督学习 vs 无监督学习】
├─ 监督学习：有标记（分类、回归）
└─ 无监督学习：无标记（聚类）
```

```
🔄 泛化相关术语

【泛化能力 Generalization】
├─ 学得模型对新样本的适应能力
└─ 核心目标：希望泛化能力强

【独立同分布 i.i.d.】
├─ Independent and Identically Distributed
└─ 假设：训练样本和测试样本来自同一分布
```

### Day 3（周三）：假设空间
**时长**：60 分钟 | **对应**：西瓜书 1.3 节

#### 🎬 学习内容
| 时间段 | 内容 | 资源 |
|--------|------|------|
| 25分钟 | 观看视频 | 视频 P1 假设空间部分 |
| 30分钟 | 阅读教材 | 西瓜书 1.3 节「假设空间」(第5-8页) |
| 5分钟 | 画图理解 | 画出假设空间示意图 |

#### 📝 核心要点（西瓜书 1.3）
```
🧠 归纳与演绎

【归纳 Induction】
├─ 从特殊到一般
├─ 从具体事实归纳出一般规律
└─ 机器学习 = 归纳学习

【演绎 Deduction】
├─ 从一般到特殊
└─ 从公理推出定理

类比：
- 归纳：看了100个西瓜，总结出"青绿+蜷缩=好瓜"
- 演绎：知道"青绿+蜷缩=好瓜"，判断新西瓜
```

```
📐 假设空间 Hypothesis Space

假设空间 = 所有可能的假设组成的集合

西瓜例子（只考虑色泽、根蒂、敲声三个属性）：

色泽 ∈ {青绿, 乌黑, 浅白, *}     (* 表示任意值)
根蒂 ∈ {蜷缩, 稍蜷, 硬挺, *}
敲声 ∈ {浊响, 沉闷, 清脆, *}

所有可能的假设数量 = 4 × 4 × 4 + 1 = 65
                                    └─ 空假设（不存在好瓜）

┌─────────────────────────────────────────────────────────┐
│  假设空间示例                                           │
│                                                         │
│  h₁: (青绿, 蜷缩, 浊响) → 好瓜                          │
│  h₂: (青绿, *, 浊响) → 好瓜                             │
│  h₃: (*, 蜷缩, *) → 好瓜                                │
│  ...                                                    │
│  h₆₅: ∅ （没有好瓜）                                    │
│                                                         │
│  学习 = 在假设空间中搜索与训练数据匹配的假设            │
└─────────────────────────────────────────────────────────┘
```

```
🎯 版本空间 Version Space

版本空间 = 与训练数据一致的假设集合

┌─────────────────────────────────────────────────────────┐
│                     假设空间                            │
│    ┌─────────────────────────────────────────────┐      │
│    │         版本空间                            │      │
│    │    （与训练数据一致的假设）                 │      │
│    │                                             │      │
│    │    h₁ ✓  h₂ ✓  h₃ ✓                        │      │
│    └─────────────────────────────────────────────┘      │
│         h₄ ✗  h₅ ✗  h₆ ✗  ...                          │
└─────────────────────────────────────────────────────────┘

学习过程：
1. 初始：版本空间 = 整个假设空间
2. 每看一个训练样本，排除不一致的假设
3. 最终：版本空间收缩到与所有训练数据一致的假设
```

### Day 4（周四）：归纳偏好 ⭐ 重点
**时长**：60 分钟 | **对应**：西瓜书 1.4 节

#### 🎬 学习内容
| 时间段 | 内容 | 资源 |
|--------|------|------|
| 25分钟 | 观看视频 | 视频 P1 归纳偏好部分 |

| 5分钟 | 思考 | NFL定理的含义 |

#### 📝 核心要点（西瓜书 1.4）
```
🤔 为什么需要归纳偏好？

问题：版本空间中可能有多个假设都与训练数据一致

例如：
h₁: (色泽=青绿) ∧ (根蒂=蜷缩) → 好瓜
h₂: (色泽=青绿) → 好瓜
h₃: (根蒂=蜷缩) → 好瓜

都能正确分类训练数据，选哪个？

┌─────────────────────────────────────────────────────────┐
│  归纳偏好 Inductive Bias                               │
│                                                         │
│  = 学习算法对某类假设的偏好                            │
│  = 在版本空间中选择假设的准则                          │
└─────────────────────────────────────────────────────────┘
```

```
✂️ 奥卡姆剃刀 Occam's Razor

"若无必要，勿增实体"

在机器学习中：
└─ 若有多个假设与数据一致，选择最简单的那个

为什么？
├─ 简单假设泛化能力更好
├─ 复杂假设可能过度拟合训练数据
└─ 简单 ≠ 短，而是"假设的属性更少/约束更宽松"

⚠️ 注意：
"简单"的定义本身就是一种偏好！
```

```
🏆 NFL 定理（没有免费午餐定理）

No Free Lunch Theorem

┌─────────────────────────────────────────────────────────┐
│  核心结论：                                             │
│                                                         │
│  脱离具体问题，空谈"哪个学习算法更好"是没有意义的！    │
│                                                         │
│  对于所有可能的问题（均匀分布）：                       │
│  任何两个学习算法的期望性能都相同                       │
└─────────────────────────────────────────────────────────┘

直观理解：
- 算法 A 在问题集 P₁ 上好，在 P₂ 上差
- 算法 B 在问题集 P₁ 上差，在 P₂ 上好
- 平均来看，两者一样

实际意义：
├─ 没有"万能"的学习算法
├─ 不同问题需要选择适合的算法
├─ 算法的优劣取决于具体问题
└─ 归纳偏好需要与问题匹配
```
