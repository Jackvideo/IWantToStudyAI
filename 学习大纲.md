## 第二阶段：神经网络核心原理 (1周) ⚡ 快速版

### 🎯 学习目标
- ✅ 理解神经网络基本结构（已完成）
- ⚡ **快速过** CNN/RNN（理解核心思想，各30分钟）
- ⭐ **重点掌握** Attention + Transformer（为 LLM 打基础）

### 📚 核心概念

#### Day 1-2：神经网络基础 ✅ 已完成
- ✅ **神经元 + MLP**：加权求和 → 激活函数 → 输出
- ✅ **前向传播 + 反向传播**：数据前向流动，梯度反向传播
- ✅ **激活函数**：Sigmoid、ReLU、Softmax
- ✅ **词嵌入 + Word2Vec**：文字 → 向量

#### Day 3：CNN 快速理解（30分钟）⚡ 简化
**核心思想**：
- ✅ **卷积操作**：滑动窗口提取局部特征
- ✅ **为什么有效**：图像的局部性 + 平移不变性
- ⚡ **够用就好**：知道 CNN 擅长图像处理即可
- ❌ **跳过**：池化细节、经典架构（LeNet/AlexNet）

#### Day 4：RNN 快速理解（30分钟）⚡ 简化
**核心思想**：
- ✅ **隐藏状态 = 记忆**：处理序列数据，保留历史信息
- ✅ **长程依赖问题**：RNN 记不住太长的上下文
- ⚡ **关键点**：理解"为什么需要 Attention"
- ❌ **跳过**：LSTM/GRU 的详细结构（直接学 Attention）

#### Day 5-7：Attention + Transformer ⭐ 重点
- [ ] **Attention 机制原理**
  - Query、Key、Value 的含义
  - Self-Attention：词与词之间的相关性
  - 解决长程依赖问题
- [ ] **Transformer 架构**
  - Encoder-Decoder 结构
  - Multi-Head Attention
  - Position Encoding（为什么需要位置信息）
  - Feed Forward Network
- [ ] **实战预热**
  - 使用 Hugging Face 调用预训练模型
  - 理解 Tokenization（文本 → Token）

### 💡 学习要点
1. ⚡ **CNN/RNN 快速过**：理解核心思想即可（各30分钟）
2. ⭐ **Attention 是重点**：花 2-3 天深入理解
3. 🎯 **目标**：为下周学习 GPT/BERT 打好基础

### ❌ 跳过内容
- ~~CNN 的池化层、卷积核详细计算~~
- ~~LSTM/GRU 的门控机制细节~~
- ~~经典网络架构（ResNet/VGG/BERT变体等）~~

## 第三阶段：大语言模型 (3-4周) ⭐ 核心重点

### 🎯 学习目标
- 深入理解 Transformer 架构
- 掌握 LLM 的工作原理（预训练、微调、RLHF）
- 精通 Prompt Engineering
- 熟练使用各种 LLM API
- **新增**：理解 Tokenization、上下文窗口等工程细节

### 📚 核心概念

#### Week 1：Transformer 深入理解
- [ ] **Attention 机制完全掌握**
  - Self-Attention 计算过程
  - Query、Key、Value 的矩阵运算
  - 为什么 Attention 能捕获长程依赖
  - Attention 的复杂度问题
- [ ] **Transformer 完整架构**
  - Encoder-Decoder 详解
  - Multi-Head Attention（为什么需要多头）
  - Position Encoding（绝对位置 vs 相对位置）
  - Feed Forward Network + Layer Norm
  - 残差连接的作用
- [ ] **关键论文理解**
  - 《Attention Is All You Need》精读
- [ ] **实战练习** 🆕
  - 使用 Hugging Face Transformers 调用 BERT/GPT
  - 理解 model.config 中的参数

#### Week 2：LLM 原理与训练
- [ ] **Tokenization 深入** 🆕
  - BPE（Byte Pair Encoding）
  - WordPiece、SentencePiece
  - Token vs 字符 vs 词
  - 不同模型的 Tokenizer 差异
- [ ] **预训练与微调**
  - 预训练任务：下一个词预测(Next Token Prediction)
  - 为什么"预测下一个词"能学到语言知识
  - 微调(Fine-tuning) vs 提示学习(Prompt Tuning)
  - LoRA、P-Tuning 等高效微调方法（概念）
- [ ] **GPT 系列演进**
  - GPT-1 → GPT-2 → GPT-3 → GPT-4
  - 涌现能力(Emergent Abilities)
  - 规模定律(Scaling Laws)
  - GPT vs BERT 的区别（单向 vs 双向）
- [ ] **RLHF（人类反馈强化学习）**
  - 为什么需要 RLHF
  - SFT（监督微调）→ Reward Model → PPO 流程
  - InstructGPT / ChatGPT 的训练过程
- [ ] **上下文窗口理解** 🆕
  - Context Length（4K/8K/32K/128K）
  - Token 计数与成本计算
  - 长上下文的挑战

#### Week 3：Prompt Engineering 精通
- [ ] **Prompt 的本质**
  - 上下文学习(In-Context Learning, ICL)
  - Few-shot、Zero-shot、One-shot
  - Chain-of-Thought（思维链）
  - 为什么 LLM 能从示例中学习
- [ ] **Prompt 设计技巧**
  - 角色设定（System Message）
  - 任务描述的最佳实践
  - 格式约束（JSON Mode、Structured Output）
  - 分步推理引导
  - 示例的选择与排序
- [ ] **高级 Prompt 技巧**
  - Self-Consistency（多次采样投票）
  - Tree of Thoughts（思维树）
  - ReAct（推理+行动）
  - 元提示（Meta-Prompting）
- [ ] **检索增强生成(RAG)预览**
  - RAG 的基本流程
  - 为什么需要 RAG（知识更新、减少幻觉）
- [ ] **实战练习** 🆕
  - 设计 10+ 不同场景的 Prompt
  - 对比不同 Prompt 策略的效果

#### Week 4：LLM API 实战与应用
- [ ] **OpenAI API 完整掌握**
  - Chat Completions API
  - Function Calling（为 Agent 做准备）
  - Embeddings API（为 RAG 做准备）
  - 流式输出(Streaming)
  - 参数详解：temperature、top_p、max_tokens、frequency_penalty 等
- [ ] **其他模型 API**
  - Claude（Anthropic）
  - 文心一言、通义千问等国内大模型
  - API 的通用模式与差异
- [ ] **Token 管理与成本优化** 🆕
  - Token 计数工具（tiktoken）
  - 成本计算与优化策略
  - 缓存机制（如 Prompt Caching）
- [ ] **常见应用模式**
  - 文本生成、摘要、翻译
  - 代码生成与代码理解
  - 问答系统、对话系统
  - 结构化输出（JSON Mode）
- [ ] **局限性与解决方案**
  - 幻觉(Hallucination)问题 → RAG
  - 上下文长度限制 → 分块策略
  - 知识截止日期 → 实时检索
  - Prompt Injection 安全问题
- [ ] **综合实战项目** 🆕
  - 构建一个完整的 LLM 应用（如智能客服、文档助手）
  - 从需求分析到部署

### 💡 学习要点
1. **Transformer 是基础**：花足够时间理解，后面都是应用
2. **理解"下一个词预测"的强大**：预训练的本质
3. **Prompt Engineering 是核心技能**：大量实战练习
4. 🆕 **工程细节很重要**：Tokenization、成本、性能优化

### 🎯 学习目标

- 掌握 Agent 开发的核心框架

### 📚 核心概念

#### Week 1：Agent 基础

- [ ] **Agent 核心组件**
  - 规划(Planning)：任务分解、反思
  - 记忆(Memory)：短期/长期记忆
  - 工具使用(Tool Use)：扩展能力边界
- [ ] **ReAct 模式**
  - Reasoning + Acting
  - 思考-行动-观察循环

#### Week 2：Agent 开发实践
- [ ] **LangChain 框架**
  - Chain、Agent、Tool 概念
  - 常用组件和使用方法
  - 构建简单的对话 Agent
- [ ] **Function Calling**
  - OpenAI Function Calling 机制
  - 工具定义和调用流程
  - 自定义工具开发
- [ ] **RAG 深入**
  - 向量数据库选择
  - 文档切分策略
  - 检索优化技巧

#### Week 3：高级 Agent 开发
- [ ] **多 Agent 系统**
  - Agent 间通信和协作
  - 任务分配和协调
  - AutoGen、CrewAI 等框架
- [ ] **Agent 设计模式**
  - 规划器-执行器模式
  - 监督者模式
  - 辩论/讨论模式
- [ ] **实战项目**
  - 构建一个完整的 Agent 应用
  - 从需求分析到部署上线

### 💡 学习要点
1. Agent 是 LLM 能力的放大器
2. 重点掌握工具使用和 RAG
3. 通过实践项目巩固所学
